# 平台总体架构

## 定位

本系统是“工程项目全过程动态造价管理信息系统”的统一整合平台，不是 25 套互不关联的小系统。平台负责公共账户、项目主数据、统一检索、附件挂接、操作留痕和数据库结构，各学生小组在同一基座上认领不同业务任务。

## 目标

- 以同一个脱敏工程项目为业务背景。
- 支持两个班约 90 名学生按组协作开发。
- 每组认领 1 个任务，并完成读、写、改、查和规则处理的完整闭环。
- 最终所有任务都能在同一平台中被查询、联动和集成展示。

## 架构分层

### 1. 表现层

- 登录页
- 首页
- 项目总览
- 全过程看板
- 生命周期导航
- 统一展示页
- 任务中心
- 帮助说明
- 后续各业务列表页、详情页、录入页、统计页

### 2. 应用层

- 认证与身份识别
- 任务分配与分组信息
- 表单保存与修改
- 统一查询条件
- 规则校验与预警
- 统一索引登记

### 3. 数据层

- 公共基础表
- 业务主线表
- 统一索引表
- 附件表
- 操作日志表

## 统一索引机制

平台必须设置统一索引表 `form_index`。任何小组新增的业务记录，只要是平台内正式表单，都要同步生成一条索引记录。统一索引至少支持以下检索入口：

- 按项目查询全部业务单据
- 按合同查询相关计量、支付、变更、结算
- 按业务类型查询同类表单
- 按状态查询待处理事项
- 按创建时间或更新时间查询最近事项

## 小组开发边界

- 每组只开发一个认领任务。
- 每组只修改允许范围内的目录和表结构。
- 每个任务至少包含 1 个主表和 1 个关联表或明细表。
- 每个任务必须实现读取、新增、修改、查询。
- 每个任务必须至少实现 1 条业务规则。
- 每个任务必须接入 `form_index`。

## 公共能力清单

- 项目主数据
- 组织与参建单位
- 学生分组与任务主题
- 表单统一索引
- 全局 TypeScript 类型
- 全局 CSS 样式入口
- 统一状态字段
- 附件关联
- 操作日志
- 公共筛选条件
- 公共字典数据

## 集成原则

- 不允许各组自行设计孤立的数据库命名体系。
- 不允许各组绕开统一索引直接做独立检索。
- 不允许各组在前端直接访问 SQLite。
- 不允许各组修改公共导航、看板、展示页和全局样式入口。
- 不允许只做静态页面，不实现最小数据处理闭环。
